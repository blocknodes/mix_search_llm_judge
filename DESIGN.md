# 检索质量评估系统 - 架构设计

## 1. 系统目标

构建一个自动化的检索质量评估 Pipeline，用于衡量混合检索系统的效果。通过 LLM 作为评判者（LLM-as-a-Judge）对检索结果进行相关性评分，计算标准化的 Hit@N 指标。

## 2. 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        评估 Pipeline                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐ │
│   │ Benchmark│───▶│ 混合检索 │───▶│ LLM 评分 │───▶│ 指标计算 │ │
│   │  数据集  │    │   模块   │    │   模块   │    │   模块   │ │
│   └──────────┘    └──────────┘    └──────────┘    └──────────┘ │
│        │               │               │               │        │
│        ▼               ▼               ▼               ▼        │
│   Query + Cat     Top-K 候选     相关性分数      Hit@1/3/10    │
│                                   (0-10)                        │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 核心模块

### 3.1 混合检索模块

- **输入**：用户 Query
- **输出**：Top-K 候选文档列表
- **能力**：
  - 向量检索 + 关键词检索融合
  - 可配置检索策略（精确/快速）
  - 自动重试与退避机制

### 3.2 LLM 评分模块

- **输入**：Query + 候选文档
- **输出**：0-10 分的相关性评分
- **评分维度**：
  - 主体一致性
  - 事实/数据支持
  - 语义相关性
  - 产品参数匹配度
- **能力**：
  - 多 LLM 后端支持
  - 结构化输出解析
  - 失败重试

### 3.3 指标计算模块

- **输入**：带评分的候选列表
- **输出**：Hit@N 指标
- **计算逻辑**：
  - 设定相关性阈值（默认 ≥8 分视为相关）
  - Hit@N = 前 N 个结果中是否存在相关文档
  - 支持 Hit@1、Hit@3、Hit@10 等多级指标

## 4. 设计决策

### 4.1 LLM-as-a-Judge

采用 LLM 作为相关性评判者，相比人工标注：
- 成本低、速度快
- 评判标准一致
- 可大规模自动化

### 4.2 多线程并发

- 检索和 LLM 调用均为 IO 密集型
- 采用线程池并发处理
- 每个线程独立的客户端实例，避免竞争

### 4.3 重试机制

- 指数退避 + 随机抖动
- 避免瞬时故障导致整体失败
- 可配置最大重试次数和延迟参数

### 4.4 配置集中化

- 所有参数（API、阈值、并发数等）集中管理
- 便于不同环境切换
- 支持多 LLM 后端配置

## 5. 数据模型

### 输入

| 字段 | 说明 |
|------|------|
| query | 用户查询文本 |
| category | 查询类别（可选） |

### 中间结果

| 字段 | 说明 |
|------|------|
| candidates | 检索候选列表 |
| candidates[].score | 检索分数 |
| candidates[].llm_relevance | LLM 相关性评分 (0-10) |

### 输出指标

| 指标 | 说明 |
|------|------|
| Hit@1 | 第 1 个结果是否相关 |
| Hit@3 | 前 3 个结果中是否有相关 |
| Hit@10 | 前 10 个结果中是否有相关 |

## 6. 扩展性

- **新增 LLM 后端**：配置文件添加即可
- **调整评分标准**：修改 Prompt 模板
- **新增指标**：扩展指标计算模块
- **不同检索源**：替换检索客户端实现
